# Техническое задание: Разработка демонстратора библиотеки tqdm

## 1. Наименование проекта
**Разработка программы-демонстратора возможностей библиотеки tqdm для визуализации прогресса выполнения задач в консольных приложениях Python**

## 2. Основание для разработки
Проект выполняется в рамках дисциплины «Технология разработки программного обеспечения» с целью исследования библиотеки tqdm и практического применения её возможностей для улучшения пользовательского опыта в консольных приложениях.

## 3. Назначение разработки
Программа предназначена для:
- Демонстрации ключевых возможностей библиотеки tqdm
- Исследования поведения прогресс-баров в различных сценариях
- Обучения работе с визуализацией прогресса в Python
- Предоставления наглядных примеров использования tqdm

## 4. Требования к программе

### 4.1. Функциональные требования

#### 4.1.1. Основное меню
Программа должна предоставлять интерактивное меню со следующими пунктами:
1. **Обработка файлов** — демонстрация базового прогресс-бара
2. **Загрузка из сети** — демонстрация вложенных прогресс-баров
3. **Обработка данных** — демонстрация ручного управления прогрессом
4. **Управление хранилищем** — просмотр и управление файлами
5. **Выход** — завершение программы

#### 4.1.2. Сценарий 1: Обработка файлов
- Генерация 20 тестовых файлов со случайным содержимым
- Расширения файлов: .txt, .log, .dat, .csv, .tmp
- Отображение прогресса обработки с помощью tqdm
- Подсчет успешных и ошибочных операций
- Сохранение обработанных файлов в отдельную директорию
- Автоматическая очистка временных файлов

#### 4.1.3. Сценарий 2: Загрузка из сети
- Имитация загрузки 5 файлов разного размера
- Реализация вложенных прогресс-баров (общий + по каждому файлу)
- Динамическое отображение скорости загрузки
- Имитация случайных задержек и проблем сети

#### 4.1.4. Сценарий 3: Обработка данных
- Генерация потока данных через генератор
- Ручное управление прогрессом через update()
- Вычисление математических функций (sqrt, log, sin, cos)
- Отображение кастомных метрик (min/max/avg время)

#### 4.1.5. Управление хранилищем
- Просмотр содержимого всех директорий
- Удаление отдельных файлов
- Очистка временной директории
- Перемещение файлов между директориями
- Копирование файлов
- Создание ZIP-архивов
- Поиск файлов по имени

### 4.2. Требования к интерфейсу
- Цветное оформление консоли
- Понятная навигация по меню
- Информативные сообщения о ходе выполнения
- Корректное отображение прогресс-баров
- Поддержка русского языка в интерфейсе

### 4.3. Требования к надежности
- Обработка некорректного ввода пользователя
- Обработка прерывания программы (Ctrl+C)
- Восстановление после ошибок ввода-вывода
- Сохранение данных при аварийном завершении

### 4.4. Требования к составу и параметрам технических средств
- Операционная система: Windows/Linux/macOS
- Процессор: любой
- ОЗУ: от 512 МБ
- Дисковое пространство: от 100 МБ

### 4.5. Требования к программной совместимости
- Python версии 3.8 или выше
- Библиотека tqdm версии 4.66.1
- Библиотека colorama версии 0.4.6 (опционально)
